Карточка займа — v2 (по скиллам)

Применены скиллы:
  • ui-ux-pro-max — метрики первыми; touch ≥44px; focus-visible; mobile-first, без horizontal scroll; сумма не обрезается.
  • frontend-design — сдержанный utilitarian/fintech тон; спокойный CTA; без кричащего primary на каждой карточке.
  • web-design-guidelines — tabular-nums, семантика кнопок/ссылок, контраст muted/text.

Исправления: hero-сумма на всю ширину (stress: 100 000 000 ₽), срок+ставка компактно, доп. поля структурированы, заёмщик внизу, CTA не доминирует в ленте.

A — hero-сумма, теги, ghost-кнопка

  • Плюс: любая длина суммы на отдельной строке; лента спокойная; быстрый скан «сколько / срок / ставка».
  • Минус: карточка +~12px по высоте; цель займа — 2 строки max; ghost слабее primary по CTR.
  • Когда: длинный список, приоритет сравнения условий (рекомендуется для 100M+).
